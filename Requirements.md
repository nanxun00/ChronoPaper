本文档为项目最终定稿业务需求，仅包含功能定义、业务流程、交互规范、能力边界。不含架构设计、不含权限细节、不含部署方案。所有功能均为 V1 正式交付功能，所有未提及功能默认不做。

## 一、系统整体定位

面向十几人科研团队的论文智能化管理、时序溯源、AI精读、个人科研Ideas管理、LaTeX论文编译一体化系统。

核心特色：**TimeRAG 时序领域时间线溯源 + 图文双向量检索 + arXiv 公私分层资源体系 + 内置 MCP 全能力集成 + LaTeX 云端编译**。

## 二、数据资源分层规则（全功能底层基准）

系统所有资源分为两层，所有功能严格遵循此分层：

### 1. 全局公共资源层

所有管理员抓取、用户个人抓取的 arXiv 论文：元数据、全文文本、文本向量、图表文本向量全员共享、全员可检索。

### 2. 用户私有资源层

用户本地上传 PDF、用户 LaTeX 项目、用户编译生成 PDF、用户私有向量、用户 Ideas、对话记录全部私有隔离。

## 三、arXiv 论文抓取系统（完整功能）

唯一数据源：arXiv，不接入其他网站。

- 全局去重：以 arxiv_id 唯一去重，已存在论文不再下载、解析、向量化
- 支持两类抓取任务：管理员全局批量任务、用户个人自定义定时任务
- 用户可自定义：arXiv 分类、关键词、每日执行时间
- 所有任务支持手动立即执行、定时自动执行
- 抓取全量日志记录：新增数量、失败原因、执行时间
- 内置限速、熔断、单日全局上限、单用户上限，防 arXiv 封禁
- 抓取完成自动下载 PDF、自动进入后台异步解析流程

## 四、PDF 版面分层解析 + 图文双向量检索系统（行业标准顶配）

系统所有 PDF（arXiv论文、用户私有PDF、LaTeX编译PDF）统一采用：**PyMuPDF + pdfplumber 版面感知分层解析策略**

### 1. 版面分层解析能力

自动识别页面区块：标题、正文、段落、表格、图片、公式、脚注

每块携带完整溯源元数据：页码、页面坐标 bbox、区块类型

解析输出：

- 纯结构化文本段落
- 表格自动转为 Markdown 结构化表格
- 公式自动识别转换为 LaTeX 字符串
- 图片单独提取保存，绑定图注、前后上下文文本

RAG 溯源全局强制开启：所有 AI 回答自带页码、原文片段、区块定位，杜绝幻觉。

### 2. 图文双向量索引体系（正式纳入需求）

采用「文本向量库 + 独立图像向量库」双索引结构

#### （1）文本向量层（复用原有 TimeRAG 全局向量库）

所有图片的图注、上下文文本生成文本向量，参与全局语义检索、TimeRAG 时序检索、AI 问答。

用户文字提问可直接检索到对应图表及其原文上下文。

#### （2）图像向量层（新增独立图片向量集合）

使用 CLIP/SigLIP 轻量视觉模型生成图片向量，独立库存放。

每条图像向量绑定完整关联字段：arxiv_id、页码、bbox、图片路径、图注文本、对应文本块ID。

#### （3）双检索模式

- **文字搜图**：自然语言描述图表 → 匹配文本向量 → 返回对应论文图表+溯源原文
- **以图搜图**：用户上传任意架构图/实验图 → 匹配图像向量库 → 返回相似图表论文列表

#### （4）图片数据库固定结构

img_id, arxiv_id, page_num, bbox, img_file_path, caption_text, text_chunk_id, image_embedding

## 五、文献检索 + AI 问答系统

- 双检索模式：关键词检索 + 语义向量检索
- 检索范围可切换：仅公共论文 / 公共+本人私有文档
- 检索结果展示：标题、作者、摘要、年份、相关度分数
- 支持单篇/多篇论文联合 AI 精读问答
- 所有问答强制带 PDF 溯源：页码、原文片段、区块位置
- 论文收藏、个人收藏库管理

## 六、TimeRAG 时序领域时间线溯源（核心特色功能）

替代传统简单演进链，采用**时序感知 RAG 多轮深挖**机制，解决时间线错乱、缺失里程碑、无因果链路问题。

### 1. 核心能力

- 支持自定义时间区间、限定 arXiv 分类筛选文献
- 时序加权检索：优先匹配对应年代文献，避免时间错乱
- 自动识别时序缺口，多轮子查询补全缺失阶段论文
- 自动划分四大阶段：奠基期、发展期、爆发期、前沿期
- 自动区分：主线技术、并行分支、淘汰技术
- 自动梳理技术迭代因果关系（提出→改进→分支→淘汰）

### 2. 标准输出内容

- 年度里程碑论文列表
- 分阶段领域发展总结
- 完整技术迭代因果链路
- Mermaid 可视化时间路线图
- 个人 Idea 时序定位分析（领域空白、创新点、竞品工作）

### 3. 配套机制

- 相同主题+时间区间 24 小时缓存，新论文入库自动失效
- 异步后台任务，支持进度查看、手动取消
- 支持 Markdown 完整报告下载
- 自动提取里程碑论文核心图表插入时间线报告

## 七、个人 Idea 智能匹配系统

- 用户可录入多条个人研究 Idea、技术思路、创新方案
- 每日论文更新后自动批量计算 0~100 匹配分数
- 首页论文自动按匹配分降序，分为高/中/低三栏
- 支持手动全局重新匹配评分
- Idea 可绑定笔记、绑定 LaTeX 项目
- TimeRAG 自动对比用户 Idea，给出时序创新定位与空白分析

## 八、LaTeX 完整上传管理 + 云端 PDF 编译系统

### 1. LaTeX 项目管理

- 支持上传 .tex、.bib、图片、cls、sty 全套项目文件
- 以独立项目文件夹隔离存储
- 支持查看源码、增删文件、覆盖更新
- 支持下载完整项目压缩包 / 单独下载编译后 PDF
- 支持项目标签、备注、关联论文与 Idea

### 2. 云端异步编译

- 支持多编译引擎，自动多轮编译处理参考文献
- 实时输出编译日志、警告、报错行号
- 编译成功自动生成 PDF
- 编译生成 PDF 自动进入用户私有文档库、自动多模态解析、私有向量入库

### 3. BibTeX 联动能力

- 论文详情页一键导出标准 BibTeX
- 批量多选论文批量导出 BibTeX
- 一键导入 BibTeX 条目至个人 LaTeX 项目参考文献文件

## 九、内置源码 MCP 能力（无独立进程）

MCP 完全源码内置，不独立部署、不跨进程通信，与 Web 功能 100% 能力对齐。

MCP 可调用全部核心功能：

- 获取每日更新论文
- 关键词/语义检索、图文检索、以图搜图
- 带溯源 RAG 问答
- TimeRAG 时序时间线生成
- 读写个人收藏、Idea、笔记
- 读取 LaTeX 项目、触发云端编译、下载 PDF
- 获取论文 BibTeX 引用

## 十、LLM 全局调度系统

- 兼容市面所有主流大模型 API + 本地开源量化模型
- 全局并发限制、接口熔断、失败重试
- RAG、TimeRAG 报告开启智能缓存，减少重复消耗

## 十二、自建知识库与 Milvus 向量检索

用户可创建多个独立知识库，每个知识库对应 Milvus 向量集合与本地元数据（`database.json` 体系）。

- 知识库列表：查看名称、描述、文档数量、创建时间
- 创建/删除知识库：支持自定义名称与描述
- 文档上传：支持 PDF、TXT 等常见格式，自动分块、向量化入库
- 文档管理：查看、删除单篇文档
- 检索测试：指定知识库进行语义检索，返回相关片段与分数
- 与对话联动：对话页可选择已建知识库作为 RAG 检索范围

后端 API：`/data/*`；实现位于 `services/rag/` + `api/v1/knowledge_base.py`。

## 十三、知识图谱（Neo4j）

基于知识库文档自动或手动构建实体关系图谱，支持可视化浏览与维护。

- 从文档抽取或上传三元组/结构化数据写入 Neo4j
- 图谱列表与单库图谱查询
- 节点/关系浏览、搜索、删除
- 前端图谱可视化（力导向图等交互）
- 与知识库文档联动：按文件名删除对应图谱数据

后端 API：`/data/graph/*`；图谱存储为 Neo4j，配置见环境变量 `NEO4J_*`。

## 十四、SmartBI 智能图表

基于选定知识库的 RAG 检索结果，由 LLM 生成结构化数据并渲染图表。

- 选择目标知识库作为数据源
- 输入自然语言分析目标（如「统计各年份论文数量」）
- 指定图表类型（柱状图、饼图等）与图表标题
- 返回 ECharts 等前端可渲染的图表配置 JSON
- 支持上传 CSV/表格类文件辅助分析（若接口已开放）

后端 API：`/smartbi/*`；前端页面 `/smartbi`。

## 十五、文档处理工具

面向预处理与调试的独立工具页，不替代第四章正式 PDF 解析流水线。

- **文本分块**：输入文本或上传文件，按策略分块预览，便于 RAG 调参
- **PDF 转文本**：上传 PDF，提取纯文本供检查或导出

后端 API：`/tool/*`；前端 `/tools` 及子路由 `text-chunking`、`pdf2txt`。

## 十六、多模态输入（语音与图片 OCR）

增强对话与检索的输入方式，与第五章 AI 问答配合使用。

### 1. 语音输入

- 前端录音或上传音频文件
- 后端调用 DashScope 多模态接口转写为文本
- 转写结果作为用户消息送入对话流程

后端 API：`/audio/*`；依赖 `DASHSCOPE_API_KEY`。

### 2. 图片 OCR

- 对话中上传图片（JPEG/PNG 等）
- 后端 OCR 识别图中文字，缓存会话级上下文
- 下一轮提问自动附带 OCR 文本作为提示词上下文（类似主流助手 图+文 问答）

后端 API：`/image/*`；实现基于 Tesseract OCR。

## 十七、系统配置与运维

面向管理员与高级用户的运行时配置能力。

- 读取/更新全局配置（模型列表、API 地址、检索参数等）
- 查看服务端运行日志
- 健康检查与服务重启入口（受控环境）
- 前端 Debug 组件展示配置与日志摘要

后端 API：`/config`、`/log` 等（`api/v1/system.py`）；前端 `SettingView` + `DebugComponent`。
